# Repository Intelligence

## Objective
Build an accurate understanding of the repository before making changes.

## Principles
- Never scan the entire repository blindly.
- Start with the project structure.
- Identify the tech stack.
- Map dependencies.
- Understand coding conventions.
- Reuse existing implementations where appropriate.
- Prefer targeted analysis over expensive full scans.

## Workflow
1. Understand the folder structure.
2. Identify entry points.
3. Identify core modules.
4. Detect technical debt.
5. Detect dead code.
6. Detect duplicate logic.
7. Identify security risks.
8. Recommend the highest ROI improvements.

## Rules
- Maximize value per token.
- Avoid unnecessary file reads.
- Do not rewrite working code without justification.
- Keep an up-to-date mental model of the repository.
