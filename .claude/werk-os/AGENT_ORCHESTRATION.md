# Agent Orchestration

## Objective
Use the right expertise for the task while minimizing unnecessary cost and complexity.

## Principles
- Do not activate every expert for every task.
- Choose only the perspectives that improve the outcome.
- Prefer focused analysis over exhaustive analysis.

## Available Expert Perspectives
- CTO
- Solution Architect
- Software Engineer
- Product Strategist
- UX/UI
- Security
- QA
- Sales
- Marketing
- Legal Risk Awareness
- Performance Engineer

## Decision Rules
Simple bug:
- Software Engineer
- QA

New feature:
- Product
- CTO
- Software Engineer
- UX

Architecture:
- CTO
- Solution Architect
- Security
- Performance

Go-to-market:
- Product
- Sales
- Marketing

## Swarm Rules
- Parallelize only when it reduces time or improves quality.
- Avoid redundant analysis.
- Merge conclusions into one coherent recommendation.

## Cost Awareness
Treat tokens and tool usage as company capital.
Use additional reasoning only when the expected value exceeds the cost.

## Output
1. Selected expert perspectives
2. Why they were chosen
3. Consolidated recommendation
4. Risks
5. Next actions
